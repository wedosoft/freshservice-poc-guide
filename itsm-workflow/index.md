# ITSM 워크플로우 

> **실습 링크**: <a href="https://gsholdings.freshservice.com/ws/2/admin/automators" target="_blank">워크플로우 자동화 설정</a>

## 1. 워크플로우 개요

Freshservice에서 관리 가능한 ITSM 워크플로우:
- 현업 담당자 요청 → 현업 리더 승인 → IT담당자 할당 → IT운영팀 처리 → 결과 확인 → 만족도 측정

## 2. 설정 및 테스트 방법

| 설정 항목 | 실습 링크 | 주요 기능 |
|---------|-----------|---------|
| 사용자/그룹 관리 | <a href="https://gsholdings.freshservice.com/ws/2/admin/home" target="_blank">사용자 관리</a> | 역할, 부서, 그룹 설정 |
| 워크플로우 자동화 | <a href="https://gsholdings.freshservice.com/ws/2/admin/automators" target="_blank">자동화 설정</a> | 승인 요청, 할당 규칙, 알림 설정 |
| SLA 정책 | <a href="https://gsholdings.freshservice.com/ws/2/admin/sla_policies" target="_blank">SLA 설정</a> | 응답/해결 시간 설정 |

### 2.1 사용자 관리 구성요소

> **실습 링크**: <a href="https://gsholdings.freshservice.com/ws/2/admin/home" target="_blank">사용자 관리 화면</a>

사용자 관리 페이지에서 필요한 구성요소를 직접 설정해보세요:

| 구성요소 | 설명 | PoC 구성 권장사항 |
|---------|------|-----------------|
| 에이전트 | 서비스 데스크 담당자 관리 | 테스트용 에이전트 3-4명 추가 |
| 역할 | 권한 설정 및 관리 | 요청자, 승인자, 관리자, 처리자 역할 생성 |
| 부서 | 조직 구조 관리 | 현업부서, IT부서, IT운영팀 등 생성 |
| 요청자 그룹 | 서비스 요청자 그룹 관리 | 현업 사용자용 그룹 생성 |
| 변경자문위원회 | 변경 요청 승인 담당 | 간단한 변경관리 워크플로우용 설정 |

**PoC 진행에 필요한 최소 구성**:
- 현업 담당자 (요청자)
- 현업 리더 (승인자)
- 지주사 IT담당자 (관리자)
- IT운영팀 (처리자)

### 2.2 ITSM 워크플로우 구현 단계

> **실습 링크**: <a href="https://gsholdings.freshservice.com/ws/2/admin/automators" target="_blank">워크플로우 자동화 설정</a>

아래 단계를 따라 요청-승인-할당-처리-결과확인-만족도 측정 워크플로우를 구현합니다:

#### 1단계: 사용자 및 그룹 설정 (소요 시간: 15분)
1. 관리자 메뉴에서 '사용자 관리' 선택
2. '부서' 탭에서 다음 부서 생성:
   - 현업부서 (요청자 그룹)
   - IT부서 (관리자 그룹)
   - IT운영팀 (처리자 그룹)
3. '역할' 탭에서 다음 역할 생성:
   - 현업 담당자 (요청 생성 권한)
   - 현업 리더 (승인 권한)
   - IT 담당자 (티켓 할당 권한)
   - IT 운영팀 (티켓 처리 권한)
4. '에이전트' 탭에서 테스트용 에이전트 생성 및 역할/부서 할당

#### 2단계: 워크플로우 자동화 규칙 설정 (소요 시간: 30분)
1. 관리자 메뉴에서 '작업 흐름 자동화 도구' 선택
2. '새 규칙 추가' 버튼 클릭하여 5개의 자동화 규칙 생성:

   **규칙 1: 요청 접수 시 승인 요청**
   - 이벤트: 새 티켓 생성 시
   - 조건: 모든 티켓
   - 작업: 승인 요청 (승인자: 현업 리더)

   **규칙 2: 승인 완료 시 IT담당자 할당**
   - 이벤트: 승인 완료 시
   - 조건: 승인됨 = 예
   - 작업: 티켓 할당 (할당 대상: IT담당자)

   **규칙 3: IT담당자의 IT운영팀 할당**
   - 이벤트: 티켓 업데이트 시
   - 조건: 상태 변경됨 = 할당됨
   - 작업: 그룹 할당 (할당 그룹: IT운영팀)

   **규칙 4: 티켓 해결 시 요청자에게 알림**
   - 이벤트: 티켓 해결 시
   - 조건: 상태 = 해결됨
   - 작업: 이메일 알림 전송 (대상: 요청자, 템플릿: 해결 알림)

   **규칙 5: 해결 후 만족도 평가 요청**
   - 이벤트: 티켓 해결 후 24시간
   - 조건: 상태 = 해결됨, 만족도 = 없음
   - 작업: 만족도 평가 요청 이메일 발송

#### 3단계: 워크플로우 테스트 (소요 시간: 15분)
1. 현업 담당자로 로그인하여 새 요청 생성
   - 서비스 포털에서 '새 티켓' 버튼 클릭
   - 제목, 설명, 우선순위 등 필수 정보 입력
   - 제출 버튼 클릭
2. 현업 리더로 로그인하여 승인 처리
   - 승인 대기 중인 티켓 확인
   - '승인' 버튼 클릭하여 처리
3. IT 담당자로 로그인하여 티켓 확인 및 할당
   - 할당된 티켓 확인
   - IT운영팀으로 그룹 할당
4. IT 운영팀으로 로그인하여 티켓 처리
   - 할당된 티켓 확인
   - 티켓 상태를 '진행 중'으로 변경
   - 작업 내용 기록
   - 티켓 상태를 '해결됨'으로 변경
5. 요청자로 로그인하여 결과 확인 및 만족도 평가
   - 해결된 티켓 확인
   - 만족도 평가 작성

## 3. 주요 테스트 시나리오

> **실습 링크**: <a href="https://gsholdings.freshservice.com/support/tickets/new" target="_blank">새 티켓 생성</a>

| 시나리오 | 테스트 방법 |
|---------|------------|
| 소프트웨어 설치 요청 | 현업 담당자 요청 → 현업 리더 승인 → IT담당자 할당 → IT운영팀 처리 → 만족도 평가 |
| 하드웨어 요청 | 현업 담당자 요청 → 현업 리더 승인 → IT담당자 할당 → IT운영팀 처리 → 만족도 평가 |

### 3.1 소프트웨어 설치 요청 시나리오 상세 단계

1. **요청 단계**
   - 현업 담당자 계정으로 로그인
   - 새 티켓 생성 화면에서 제목에 "Adobe Photoshop 설치 요청" 입력
   - 설명에 "업무용 이미지 편집을 위한 Adobe Photoshop 설치가 필요합니다" 입력
   - 우선순위: 중간
   - 제출 버튼 클릭

2. **승인 단계**
   - 현업 리더 계정으로 로그인
   - 알림에서 승인 요청 확인
   - 승인 사유 입력: "업무상 필요성 확인됨"
   - 승인 완료

3. **할당 및 처리 단계**
   - IT 담당자 계정으로 로그인
   - 승인된 티켓 확인
   - IT운영팀 할당
   - IT운영팀 계정으로 로그인
   - 티켓 상태를 '진행 중'으로 변경
   - 내부 노트에 "라이선스 확인 및 설치 작업 진행 중" 입력
   - 티켓 상태를 '해결됨'으로 변경
   - 해결 노트에 "Adobe Photoshop CC 2023 설치 완료" 입력

4. **만족도 평가 단계**
   - 현업 담당자 계정으로 로그인
   - 해결된 티켓 확인
   - 만족도 평가 입력 (5점 만점에 4점)
   - 코멘트 입력: "신속한 설치 감사합니다"

## 4. 확인 포인트

- 자동 알림 발송 여부
- 각 단계별 상태 변경 적용
- SLA 타이머 작동 확인
- 워크플로우 보고서 생성

### 4.1 실행 결과 확인사항

- **자동화 규칙 작동**: 각 단계에서 자동화 규칙이 올바르게 실행되었는지 확인
- **이메일 알림**: 이해관계자에게 적절한 알림이 발송되었는지 확인
- **티켓 상태 추적**: 전체 워크플로우 과정에서 티켓 상태가 적절히 변경되었는지 확인
- **감사 추적**: 티켓 기록에 모든 변경 내역과 액션이 기록되었는지 확인
- **보고서**: 워크플로우 전체 과정이 보고서에 정확히 반영되는지 확인
