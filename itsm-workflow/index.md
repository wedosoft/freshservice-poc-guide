# ITSM 워크플로우 템플릿

## 1. 워크플로우 개요

### 목적
GS의 IT 서비스 관리 프로세스를 Freshservice를 통해 자동화하고, 현업 담당자 요청부터 만족도 측정까지의 전체 프로세스를 효율적으로 관리하기 위한 워크플로우입니다.

### 워크플로우 흐름
1. 현업 담당자 요청
2. 현업 리더 승인
3. 지주사 IT담당자 확인 및 할당
4. 할당받은 IT운영팀 확인 및 처리
5. 처리 완료 결과 확인
6. 만족도 측정

## 2. 티켓 상태 및 역할 정의 (아래는 예시입니다.)

### 티켓 상태
- **신규(New)**: 요청이 처음 접수된 상태
- **승인 대기(Pending Approval)**: 현업 리더 또는 IT담당자의 승인을 기다리는 상태
- **승인됨(Approved)**: 승인이 완료된 상태
- **할당 대기(Pending Assignment)**: IT담당자의 할당을 기다리는 상태
- **할당됨(Assigned)**: IT운영팀에 할당된 상태
- **진행 중(In Progress)**: IT운영팀이 처리 중인 상태
- **해결됨(Resolved)**: 처리가 완료된 상태
- **종료(Closed)**: 만족도 평가까지 완료된 상태
- **거부됨(Rejected)**: 승인이 거부된 상태

### 역할 정의
- **현업 담당자(Requester)**: 서비스 요청을 생성하는 사용자
- **현업 리더(Department Manager)**: 현업 담당자의 요청을 승인하는 관리자
- **지주사 IT담당자(IT Manager)**: 요청을 검토하고 IT운영팀에 할당하는 담당자
- **IT운영팀(IT Agent)**: 실제 요청을 처리하는 기술 담당자

## 3. 구현 단계

### 3.1 기본 설정

#### 관리자 계정 설정
1. 관리자 > 계정 설정에서 기본 정보 확인
2. 관리자 > 사용자 관리에서 필요한 사용자 계정 생성
3. 관리자 > 그룹에서 다음 그룹 생성:
   - 관리자 그룹
   - 보고 관리자 그룹 (현업 리더)
   - 서비스 요청 처리팀 (IT운영팀)

#### 티켓 상태 설정
1. 관리자 > 티켓 > 상태에서 다음 상태 생성:
   - 신규(New): 티켓 생성 시 초기 상태
   - 승인 대기(Waiting for Approval): 승인 요청 발송 후 상태
   - 할당 대기(Waiting for Assignment): 승인 완료 후 담당자 할당 전 상태
   - 진행 중(In Progress): 담당자가 작업 중인 상태
   - 해결됨(Resolved): 작업 완료 후 상태
   - 거부됨(Rejected): 승인자가 요청을 거부한 상태
   - 종료(Closed): 요청자 확인 후 티켓 종료 상태

### 3.2 서비스 카탈로그 설정

#### 서비스 카탈로그 구성
1. 관리자 > 서비스 카탈로그 > 카테고리에서 다음 카테고리 생성:
   - 소프트웨어: 소프트웨어 관련 요청
   - 하드웨어: 하드웨어 관련 요청
   - 계정/권한: 계정 및 접근 권한 관련 요청
   - 문제 해결: 일반적인 IT 문제 해결 요청
2. 각 카테고리 아래 서비스 항목 생성:
   - 소프트웨어 > Adobe Premiere Pro 설치 요청
   - 소프트웨어 > 고급 디자인 소프트웨어 요청
   - 하드웨어 > Windows 노트북 요청
   - 하드웨어 > 하드웨어 문제 해결 요청

#### 서비스 요청 양식 설정
1. 관리자 > 서비스 카탈로그 > 서비스 항목에서 '새 서비스 항목' 클릭
2. 기본 정보 설정:
   - 서비스 이름
   - 설명
   - 카테고리 선택
3. 양식 필드 구성:
   - 요청 제목 (기본 필드)
   - 요청 설명 (기본 필드)
   - 우선순위 (드롭다운)
   - 요청 사유 (텍스트 필드)
   - 첨부 파일 (파일 업로드)
4. 워크플로우 자동화: "요청 항목 승인 워크플로우" 연결
5. 저장 및 게시

### 3.3 승인 프로세스 설정

> **팁**: 단순한 승인 프로세스로 시작하면 시스템에 쉽게 적응하고 나중에 필요에 따라 확장할 수 있습니다.

**1단계: 기본 승인 워크플로우 생성**
1. **관리자** 메뉴로 이동
2. 좌측 메뉴에서 **작업 흐름 자동화 도구** 선택
3. 오른쪽 상단의 **'+ 새 작업 흐름'** 버튼 클릭
4. 작업 흐름 이름을 **"요청 항목 승인 워크플로우"**로 입력
5. 설명에 "요청 항목에 대한 보고 관리자 승인 후 처리 프로세스" 입력

**2단계: 트리거 설정**
1. 트리거 섹션에서:
   - **이벤트**: "**티켓이 생성됨**" 선택
   - **조건 추가** 클릭
   - "**요청 유형**"이 "**서비스 요청**"일 때 선택
   - **저장** 버튼 클릭

**3단계: 승인 흐름 구성**
1. **작업** 섹션으로 이동
2. 첫 번째 노드는 "**요청 항목이 'Adobe Premiere Pro...'**"와 같이 요청된 항목 표시
3. 다음 노드 추가:
   - **'+'** 버튼 클릭
   - "**보고 관리자 승인 요청**" 선택
   - 노드 이름: "**Send for Reporting Manager approval**" 입력
   - **티켓 상태 변경**: "**승인 대기**"로 설정
   - **저장** 버튼 클릭

4. 승인 후 작업 설정 (승인됨 경로):
   - "**승인됨**" 경로에서 **'+'** 버튼 클릭
   - "**서비스 요청 처리팀에 할당**" 선택
   - 노드 이름: "**Assign to Service Request Fulfillment team**"
   - 드롭다운에서 "**서비스 요청 처리팀**" 선택
   - **티켓 상태 변경**: "**할당 대기**"로 설정
   - **저장** 버튼 클릭

5. 거부 시 작업 설정 (거부됨 경로):
   - "**거부됨**" 경로에서 **'+'** 버튼 클릭
   - "**요청자에게 알림 및 티켓 종료**" 선택
   - 노드 이름: "**Notify Requester and Close ticket**"
   - **티켓 상태 변경**: "**거부됨**"으로 설정
   - **저장** 버튼 클릭

**4단계: 활성화**
1. 화면 상단의 **저장** 버튼 클릭
2. 작업 흐름 목록에서 새로 만든 흐름의 상태를 "**활성**"으로 설정

> **참고**: 이 기본 승인 프로세스는 모든 서비스 요청에 적용됩니다. 특정 요청 항목에 따라 다른 승인 흐름이 필요한 경우 추가 조건을 설정할 수 있습니다.

### 3.4 SLA 및 알림 설정

#### SLA 정책 설정
1. 관리자 > SLA에서:
   - 새 SLA 정책 생성
   - 응답 시간 및 해결 시간 설정
   - 우선순위별 SLA 차등 적용:
     - 높음: 응답 2시간, 해결 8시간
     - 중간: 응답 4시간, 해결 16시간
     - 낮음: 응답 8시간, 해결 24시간
   - 업무 시간 정의 (평일 9시-18시)
   - 에스컬레이션 규칙 설정: SLA 위반 시 관리자에게 알림

#### 알림 설정
1. 관리자 > 알림에서:
   - 티켓 생성 알림: 요청자 및 담당 그룹
   - 승인 요청 알림: 승인자
   - 승인/거부 알림: 요청자 및 담당 그룹
   - 상태 변경 알림: 요청자
   - 해결 알림: 요청자
   - SLA 위반 알림: 담당자 및 관리자

### 3.5 자동화 규칙 설정
1. 관리자 > 작업 흐름 자동화 도구에서 다음 규칙 생성:
   - 할당 시 규칙:
     - 트리거: 티켓 할당
     - 작업: 상태 변경 (진행 중)
   - 해결 시 규칙:
     - 트리거: 티켓 해결
     - 작업: 상태 변경 (해결됨), 만족도 평가 요청
   - 종료 자동화 규칙:
     - 트리거: 해결 후 24시간 경과
     - 작업: 상태 변경 (종료)

## 4. 시연 시나리오

### 시나리오 1: 소프트웨어 설치 요청 (Adobe Premiere Pro)

#### 단계 1: 요청 생성
1. 사용자가 서비스 포털에 로그인
2. 상단의 '새 티켓' 버튼 클릭
3. 서비스 카탈로그에서 '소프트웨어' 카테고리 선택
4. 'Adobe Premiere Pro 설치 요청' 서비스 항목 선택
5. 요청 양식 작성:
   - 제목: "Adobe Premiere Pro 설치 요청"
   - 설명: "영상 편집 작업을 위해 Adobe Premiere Pro 설치가 필요합니다."
   - 우선순위: 중간
   - 요청 사유: "신규 프로젝트 영상 작업"
6. 제출 버튼 클릭
7. 시스템이 자동으로 티켓 생성 및 상태를 '승인 대기'로 변경

#### 단계 2: 보고 관리자 승인
1. 사용자의 보고 관리자가 이메일 알림 수신
2. 이메일의 링크를 통해 승인 페이지 접속
3. 요청 내용 검토
4. '승인' 버튼 클릭
5. 승인 코멘트 입력: "프로젝트에 필요한 소프트웨어로 승인합니다."
6. 시스템이 자동으로 상태를 '할당 대기'로 변경하고 서비스 요청 처리팀에 티켓 할당

#### 단계 3: 서비스 요청 처리팀 처리
1. 서비스 요청 처리팀이 할당된 티켓 확인
2. 티켓을 열어 요청 내용 검토
3. 티켓 할당 시 자동화 규칙에 의해 상태가 '진행 중'으로 변경됨
4. 소프트웨어 라이선스 확인
5. 원격 접속으로 소프트웨어 설치
6. 설치 완료 후 '상태 변경'을 클릭하여 '해결됨'으로 변경
7. 해결 메모 입력: "Adobe Premiere Pro 설치 및 라이선스 활성화 완료했습니다."
8. 시스템이 자동으로 요청자에게 해결 알림 및 만족도 평가 요청 발송

#### 단계 4: 요청자 확인 및 만족도 평가
1. 사용자가 이메일 알림 수신
2. 소프트웨어 설치 확인
3. 만족도 설문 작성:
   - 전반적인 만족도: 5점
   - 응답 시간 만족도: 4점
   - 해결 품질 만족도: 5점
   - 추가 의견: "빠른 처리 감사합니다."
4. '종료' 버튼 클릭하여 티켓 종료
5. 티켓 상태가 '종료'로 변경됨

### 시나리오 2: 요청 항목 거부 케이스

#### 단계 1: 요청 생성
1. 사용자가 서비스 포털에 로그인
2. 상단의 '새 티켓' 버튼 클릭
3. 서비스 카탈로그에서 '소프트웨어' 카테고리 선택
4. '고급 디자인 소프트웨어 요청' 서비스 항목 선택
5. 요청 양식 작성:
   - 제목: "Adobe After Effects 설치 요청"
   - 설명: "특수 효과 작업을 위한 소프트웨어가 필요합니다."
   - 우선순위: 중간
   - 요청 사유: "개인 프로젝트 작업"
6. 제출 버튼 클릭
7. 시스템이 자동으로 상태를 '승인 대기'로 변경

#### 단계 2: 보고 관리자 검토 및 거부
1. 사용자의 보고 관리자가 이메일 알림 수신
2. 이메일의 링크를 통해 승인 페이지 접속
3. 요청 내용 검토
4. '거부' 버튼 클릭
5. 거부 사유 입력: "개인 프로젝트용 소프트웨어는 회사 정책상 승인할 수 없습니다."
6. 시스템이 자동으로 상태를 '거부됨'으로 변경하고 요청자에게 거부 알림 발송

#### 단계 3: 요청자 확인
1. 사용자가 거부 알림 이메일 수신
2. 거부 사유 확인
3. 티켓이 자동으로 '거부됨' 상태로 유지됨

> 💡 **팁**: 거부된 요청에 대해 추가 정보가 필요한 경우, 사용자는 새 티켓을 생성하여 업무 관련 필요성을 더 자세히 설명할 수 있습니다.

## 5. 모범 사례 및 팁

### 5.1 구현 시 고려사항
- 자주 요청되는 서비스는 서비스 카탈로그에 표준화된 항목으로 등록
- 부서별 승인 경로를 명확히 정의하여 지연 최소화
- SLA를 우선순위별로 차등 적용하여 중요 요청 우선 처리

### 사용자 경험 개선
- 서비스 카탈로그 항목에 명확한 설명과 예상 처리 시간 표시
- 요청 상태 변경 시 자동 알림으로 투명성 확보
- 자주 묻는 질문(FAQ)과 지식 베이스 문서 제공

### 보고 및 분석
- 정기적인 SLA 준수율 모니터링
- 요청 유형별 처리 시간 분석
- 만족도 점수 추세 분석
- 반복적인 요청 패턴 식별 및 개선 방안 수립
