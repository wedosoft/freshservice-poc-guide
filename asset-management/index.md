# 자산관리 워크플로우 템플릿

## 1. 자산관리 개요

### 목적
GS의 IT 자산 수명주기를 Freshservice를 통해 효율적으로 관리하고, 자산 관련 프로세스(입사-퇴직, 자산이관, 폐기 등)를 자동화하여 자산 가시성을 높이고 비용을 최적화하기 위한 워크플로우입니다.

### 자산 수명주기
1. **자산 도입 및 등록**
   - 요구사항 정의 및 구매 요청
   - 벤더 선정 및 구매 진행
   - 자산 등록 및 초기 설정
2. **자산 할당 (신규 입사자 온보딩)**
   - 사용자 요구사항 분석
   - 적합한 자산 선택 및 준비
   - 사용자 교육 및 인계
3. **자산 사용 및 유지보수**
   - 정기 점검 및 업데이트
   - 문제 해결 및 수리
   - 성능 모니터링
4. **자산 이관 (부서 이동, 사용자 변경)**
   - 이관 요청 및 승인
   - 데이터 백업 및 초기화
   - 신규 사용자 설정
5. **자산 회수 (퇴직자 오프보딩)**
   - 회수 계획 수립
   - 데이터 백업 및 초기화
   - 상태 평가
6. **자산 폐기 및 처분**
   - 폐기 대상 식별 및 승인
   - 데이터 완전 삭제
   - 환경 친화적 처분

### 자산 수명주기 단계별 주요 작업

#### 도입 단계
- **요구사항 정의**
  - 사용자/부서 요구사항 수집
  - 기술 명세 작성
  - 예산 확인 및 승인
- **구매 프로세스**
  - 벤더 비교 및 선정
  - 구매 주문 생성
  - 배송 추적
- **초기 설정**
  - 자산 식별자 할당 (자산 태그)
  - 초기 하드웨어/소프트웨어 설정
  - 인벤토리 등록

#### 할당 단계
- **사용자 매칭**
  - 직무별 필요 자산 유형 결정
  - 사용자 선호도 고려
  - 접근 권한 설정
- **할당 프로세스**
  - 자산 준비 확인
  - 사용자 기본 교육 제공
  - 인수 확인서 작성
- **문서화**
  - 할당 정보 기록
  - 사용자 매뉴얼 제공
  - 지원 연락처 안내

## 2. 자산 데이터 구조

### 기본 정보
- **자산 태그**: 고유 식별자 (예: GS-PC-0001)
- **자산명**: 자산의 이름 (예: Dell Latitude 7420)
- **자산 유형**: PC, 노트북, 태블릿, 모니터 등
- **제조사**: Dell, HP, Apple 등
- **모델**: 제품 모델명
- **시리얼 번호**: 제조사 시리얼 번호

### 관리 정보
- **구매일**: 자산 구매 날짜
- **보증 만료일**: 보증 서비스 종료 날짜
- **상태**: 사용 중, 수리 중, 폐기 예정, 폐기 완료 등
- **할당 상태**: 할당됨, 미할당, 예약됨 등
- **위치**: 자산의 물리적 위치 (사무실, 지점 등)

### 할당 정보
- **사용자**: 자산을 사용하는 직원
- **부서**: 자산이 할당된 부서
- **할당일**: 사용자에게 할당된 날짜
- **반납 예정일**: 임시 할당의 경우 반납 예정 날짜

### 기술 정보
- **OS**: 운영체제 정보
- **CPU**: 프로세서 정보
- **RAM**: 메모리 용량
- **스토리지**: 저장 공간 정보
- **네트워크**: MAC 주소, IP 주소 등

## 4. 자산 수명주기 워크플로우

### 4.1 신규 입사자 자산 할당 워크플로우

#### 워크플로우 개요
신규 입사자에게 필요한 IT 자산을 준비하고 할당하는 프로세스입니다.

#### 워크플로우 단계
1. **입사 예정 알림**
   - 인사팀에서 신규 입사자 정보 입력
   - 입사 예정일 14일 전 자동 알림 생성

2. **자산 준비**
   - IT팀에서 필요 자산 목록 확인 (직무별 표준 구성)
   - 가용 자산 확인 또는 신규 자산 구매 요청
   - 자산 초기 설정 (OS 설치, 소프트웨어 설치, 계정 생성 등)

3. **자산 할당**
   - 자산 정보 업데이트 (할당 상태, 사용자, 부서 등)
   - 자산 인계 일정 조율

4. **자산 인계 및 교육**
   - 입사일에 자산 인계
   - 기본 사용법 및 IT 정책 안내
   - 자산 인수 확인서 서명

5. **완료 및 기록**
   - 자산 할당 완료 상태 업데이트
   - 관련 문서 첨부 및 기록

#### 구현 방법
1. 관리자 > 작업 흐름 자동화 도구에서:
   - 새 자동화 규칙 생성
   - 트리거 탭에서:
     - 이벤트: 신규 입사자 온보딩 요청 생성
   - 작업 탭에서:
     - 자산 준비 체크리스트 생성
     - 담당자 할당
     - 알림 설정
   - 저장 및 활성화

### 4.2 퇴직자 자산 회수 워크플로우

#### 워크플로우 개요
퇴직 예정 직원의 IT 자산을 회수하고 적절히 처리하는 프로세스입니다.

#### 워크플로우 단계
1. **퇴직 예정 알림**
   - 인사팀에서 퇴직 예정자 정보 입력
   - 퇴직 예정일 14일 전 자동 알림 생성

2. **자산 확인**
   - 할당된 모든 자산 목록 확인
   - 자산 회수 체크리스트 생성

3. **자산 회수**
   - 퇴직일에 모든 자산 회수
   - 자산 상태 확인 및 기록
   - 데이터 백업 및 초기화

4. **자산 상태 업데이트**
   - 자산 정보 업데이트 (할당 상태, 사용자 제거 등)
   - 자산 위치 업데이트 (IT 창고 등)

5. **자산 평가 및 처리 결정**
   - 자산 상태 평가
   - 재사용, 수리, 폐기 등 처리 방향 결정

#### 구현 방법
1. 관리자 > 작업 흐름 자동화 도구에서:
   - 새 자동화 규칙 생성
   - 트리거 탭에서:
     - 이벤트: 퇴직자 오프보딩 요청 생성
   - 작업 탭에서:
     - 자산 회수 체크리스트 생성
     - 담당자 할당
     - 알림 설정
   - 저장 및 활성화

### 4.3 자산 이관 워크플로우

#### 워크플로우 개요
부서 이동이나 업무 변경 등으로 인해 자산을 다른 사용자에게 이관하는 프로세스입니다.

#### 워크플로우 단계
1. **이관 요청**
   - 부서 관리자 또는 사용자가 자산 이관 요청 생성
   - 이관 사유, 대상 사용자, 희망일 등 정보 입력

2. **요청 승인**
   - IT 관리자가 요청 검토 및 승인
   - 이관 계획 수립

3. **자산 회수 및 준비**
   - 현 사용자로부터 자산 회수
   - 필요 시 데이터 백업 및 초기화
   - 새 사용자를 위한 설정 변경

4. **자산 재할당**
   - 새 사용자에게 자산 인계
   - 기본 사용법 안내 (필요 시)
   - 자산 인수 확인서 서명

5. **완료 및 기록**
   - 자산 정보 업데이트 (사용자, 부서, 위치 등)
   - 관련 문서 첨부 및 기록

#### 구현 방법
1. 관리자 > 서비스 카탈로그에서:
   - 새 서비스 항목 생성 (자산 이관 요청)
   - 양식 필드 구성
   - 작업 흐름 자동화 규칙 설정
   - 저장 및 게시

### 4.4 자산 폐기 워크플로우

#### 워크플로우 개요
노후화되거나 손상된 자산을 안전하게 폐기하는 프로세스입니다.

#### 워크플로우 단계
1. **폐기 대상 식별**
   - 노후 자산 보고서 검토
   - 수리 불가능한 손상 자산 확인
   - 폐기 대상 목록 작성

2. **폐기 승인 요청**
   - 폐기 대상 자산 목록 및 사유 제출
   - 대체 자산 계획 포함 (필요 시)

3. **승인 프로세스**
   - IT 관리자 승인
   - 자산 관리 책임자 승인
   - 필요 시 재무 부서 승인

4. **데이터 삭제 및 초기화**
   - 저장 매체의 데이터 완전 삭제
   - 공장 초기화 또는 하드 드라이브 물리적 파괴

5. **물리적 폐기**
   - 환경 규제에 맞는 폐기 방법 선택
   - 인증된 폐기 업체 활용
   - 폐기 증명서 수령

6. **자산 기록 업데이트**
   - 자산 상태를 '폐기 완료'로 변경
   - 폐기 날짜, 방법, 증명서 번호 등 기록
   - 관련 문서 첨부

#### 구현 방법
1. 관리자 > 서비스 카탈로그에서:
   - 새 서비스 항목 생성 (자산 폐기 요청)
   - 양식 필드 구성
   - 작업 흐름 자동화 규칙 설정
   - 저장 및 게시

## 5. 구현 단계

### 5.1 자산 데이터 업로드

#### CSV 파일 준비
1. 제공된 샘플 CSV 형식에 맞게 자산 데이터 준비
2. 필수 필드 확인 (자산 태그, 자산명, 자산 유형, 제조사, 모델 등)
3. 날짜 형식 통일 (YYYY-MM-DD)

#### 한글 칼럼명 CSV 사용 가이드
1. **한글 칼럼명 사용 시 고려사항**
   - CSV 파일은 반드시 **UTF-8 인코딩**으로 저장
   - Excel에서 작업 시 "다른 이름으로 저장" → "CSV UTF-8 (쉼표로 분리)" 선택
   - 한글 칼럼명에 특수문자 사용 지양 (밑줄(_)은 사용 가능)

2. **필드 매핑 방법**
   - Freshservice는 가져오기 과정에서 한글 칼럼명을 시스템 필드에 매핑 지원
   - 매핑 화면에서 한글 칼럼명과 Freshservice 필드 연결 가능
   
3. **한글-영문 칼럼명 매핑 예시**

| 한글 칼럼명 | Freshservice 시스템 필드 |
|------------|------------------------|
| 자산_태그  | asset_tag              |
| 자산명     | name                   |
| 자산_유형  | asset_type             |
| 제조사     | manufacturer           |
| 모델       | model                  |
| 시리얼_번호 | serial_number          |
| 구매일     | acquisition_date       |
| 보증_만료일 | warranty_expiry_date   |
| 상태       | status                 |
| 할당_상태   | assigned_status        |
| 위치       | location               |
| 사용자     | assigned_to            |
| 부서       | department             |
| 할당일     | assigned_on            |

4. **한글 CSV 필드 매핑 단계**
   - **1단계: 파일 업로드 전 확인**
     - CSV 파일이 UTF-8로 올바르게 저장되었는지 확인
     - 메모장으로 열어 한글이 깨지지 않는지 확인
     - 첫 행에 컬럼명이 올바르게 포함되어 있는지 확인
   
   - **2단계: 매핑 프로세스**
     - 가져오기 화면에서 시스템이 자동으로 일부 필드 매핑 시도
     - 미매핑된 필드에 대해 드롭다운에서 적절한 시스템 필드 선택
     - 매핑이 불명확한 필드는 '매핑 안함' 선택 가능
   
   - **3단계: 매핑 검증**
     - 모든 필수 필드가 매핑되었는지 확인
     - 샘플 데이터 미리보기로 매핑 결과 확인
     - 오류가 있을 경우 매핑 조정

5. **CSV 업로드 시 주의사항**
   - 사용자 필드는 Freshservice에 등록된 사용자와 일치해야 함 (이메일 또는 사용자명)
   - 부서 필드는 시스템에 정의된 부서명과 일치해야 함
   - 자산 상태와 할당 상태는 시스템에 정의된 값과 일치해야 함
   - 날짜 형식은 YYYY-MM-DD 형식 유지

#### 업로드 절차
1. 관리자 > 자산 > 가져오기에서:
   - CSV 가져오기 선택
   - 준비된 CSV 파일 업로드
   - 필드 매핑 확인 및 조정
   - 가져오기 실행
   - 결과 확인 및 오류 수정

#### 자주 발생하는 CSV 업로드 오류 및 해결 방법

| 오류 | 원인 | 해결 방법 |
|-----|-----|---------|
| 인코딩 오류 | 잘못된 문자 인코딩 사용 | CSV를 UTF-8로 다시 저장 |
| 필드 매핑 실패 | 필드명 불일치 | 매핑 화면에서 수동 매핑 수행 |
| 사용자 불일치 | 존재하지 않는 사용자 | 시스템에 등록된 사용자명으로 수정 |
| 날짜 형식 오류 | 잘못된 날짜 형식 | YYYY-MM-DD 형식으로 통일 |
| 중복 자산 태그 | 기존 자산과 중복된 태그 | 고유한 자산 태그로 수정 |

### 5.2 자산 상태 및 관계 설정

#### 자산 상태 정의
1. 관리자 > 자산 > 자산 상태에서:
   - 필요한 상태 정의:
     - 신규(New): 자산이 등록된 초기 상태
     - 사용 중(In Use): 사용자에게 할당된 상태
     - 미사용(Available): 할당 가능한 상태
     - 수리 중(In Repair): 수리가 필요한 상태
     - 폐기 예정(To Be Disposed): 폐기 절차 진행 중인 상태
     - 폐기 완료(Disposed): 폐기가 완료된 상태

#### 관계 유형 정의
1. 관리자 > 자산 > 관계 유형에서:
   - 필요한 관계 유형 정의:
     - 연결됨(Connected to): PC와 모니터 같은 연결 관계
     - 부속품(Accessory of): 노트북과 도킹 스테이션 같은 관계
     - 하위 항목(Component of): 워크스테이션과 그래픽 카드 같은 관계

#### 자산 간 관계 설정
1. 자산 > 자산 목록에서:
   - 관계를 설정할 자산 선택
   - '관계' 탭 선택
   - '관계 추가' 버튼 클릭
   - 관련 자산 및 관계 유형 선택
   - 저장

### 5.3 서비스 카탈로그 설정

#### 자산 관련 서비스 카테고리 생성
1. 관리자 > 서비스 카탈로그 > 카테고리에서:
   - 'IT 자산 요청' 카테고리 생성
   - '자산 관리' 카테고리 생성

#### 서비스 항목 생성
1. 관리자 > 서비스 카탈로그 > 서비스 항목에서:
   - 다음 서비스 항목 생성:
     - '신규 입사자 온보딩'
     - '퇴직자 오프보딩'
     - '자산 이관 요청'
     - '자산 폐기 요청'
     - '자산 수리 요청'
   - 각 항목별 적절한 양식 필드 구성
   - 승인 흐름 및 작업 흐름 자동화 연결

### 5.4 자산 수명주기 자동화 설정

#### 온보딩 자동화 설정
1. 관리자 > 작업 흐름 자동화 도구에서:
   - 새 작업 흐름 생성
   - 이름: "신규 입사자 자산 할당 워크플로우"
   - 트리거 설정:
     - 이벤트: "티켓이 생성됨"
     - 조건: "서비스 항목"이 "신규 입사자 온보딩"일 때
   - 작업 설정:
     - 상태를 "처리 중"으로 변경
     - 서비스 요청 처리팀에 할당
     - 자산 준비 체크리스트 생성
     - 완료 시 알림 설정
   - 저장 및 활성화

#### 오프보딩 자동화 설정
1. 관리자 > 작업 흐름 자동화 도구에서:
   - 새 작업 흐름 생성
   - 이름: "퇴직자 자산 회수 워크플로우"
   - 트리거 설정:
     - 이벤트: "티켓이 생성됨"
     - 조건: "서비스 항목"이 "퇴직자 오프보딩"일 때
   - 작업 설정:
     - 상태를 "처리 중"으로 변경
     - 서비스 요청 처리팀에 할당
     - 자산 회수 체크리스트 생성
     - 완료 시 알림 설정
   - 저장 및 활성화

#### 자산 상태 변경 자동화
1. 관리자 > 작업 흐름 자동화 도구에서:
   - 새 자동화 규칙 생성
   - 이름: "자산 상태 자동 업데이트"
   - 트리거 설정:
     - 이벤트: "티켓이 해결됨"
     - 조건: 자산 관련 티켓인 경우
   - 작업 설정:
     - 온보딩 완료 시: 자산 상태를 "사용 중"으로 변경
     - 오프보딩 완료 시: 자산 상태를 "미사용"으로 변경
     - 수리 완료 시: 자산 상태를 "사용 중" 또는 "미사용"으로 변경
     - 폐기 완료 시: 자산 상태를 "폐기 완료"로 변경
   - 저장 및 활성화

#### 알림 자동화 설정
1. 관리자 > 작업 흐름 자동화 도구에서:
   - 새 자동화 규칙 생성
   - 이름: "자산 알림 자동화"
   - 트리거 설정:
     - 이벤트: 다양한 자산 관련 이벤트
   - 작업 설정:
     - 보증 만료 30일 전 알림
     - 자산 할당 시 사용자 및 관리자 알림
     - 자산 회수 7일 전 알림
   - 저장 및 활성화

## 6. 시연 시나리오

### 시나리오 1: 신규 입사자 온보딩

#### 단계 1: 온보딩 요청 생성
1. 인사 담당자가 서비스 포털에 로그인
2. 상단의 '새 티켓' 버튼 클릭
3. 서비스 카탈로그에서 'IT 자산 요청' > '신규 입사자 온보딩' 선택
4. 요청 양식 작성:
   - 신규 입사자: 김신입
   - 직급: 대리
   - 부서: 마케팅
   - 입사일: 2025-04-15
   - 필요 자산: 노트북, 모니터, 태블릿
   - 필요 소프트웨어: Office 365, Adobe Creative Cloud
5. 제출 버튼 클릭
6. 시스템이 자동으로 티켓 생성 및 상태를 '처리 중'으로 변경

#### 단계 2: 자산 준비
1. 서비스 요청 처리팀이 티켓 확인
2. 자동 생성된 체크리스트에 따라 작업 진행
3. 가용 자산 확인:
   - 노트북: GS-PC-0015 (Dell Latitude 9420) - 상태: 미사용
   - 모니터: 신규 구매 필요
   - 태블릿: 재고 확인
4. 필요 소프트웨어 설치 및 계정 생성
5. 작업 상태 업데이트: "자산 준비 완료, 인계 대기 중"

#### 단계 3: 자산 할당 및 인계
1. 입사일에 자산 인계 진행
2. 자산 정보 업데이트:
   - 할당 상태: 할당됨
   - 사용자: 김신입
   - 부서: 마케팅
   - 위치: 본사 3층
   - 할당일: 2025-04-15
3. 사용자 기본 교육 실시
4. 자산 인수 확인서 서명 수령 및 티켓에 첨부

#### 단계 4: 완료 처리
1. '상태 변경'을 클릭하여 '해결됨'으로 변경
2. 해결 메모 입력: "김신입 대리 온보딩 완료. 노트북(GS-PC-0015), 모니터(GS-MON-0015), 태블릿(GS-TAB-0008) 할당 및 소프트웨어 설치 완료."
3. 자동화 규칙에 의해 자산 상태가 '사용 중'으로 변경됨
4. 사용자가 완료 확인 후 '종료' 처리

### 시나리오 2: 퇴직자 오프보딩

#### 단계 1: 오프보딩 요청 생성
1. 인사 담당자가 서비스 포털에 로그인
2. 상단의 '새 티켓' 버튼 클릭
3. 서비스 카탈로그에서 'IT 자산 요청' > '퇴직자 오프보딩' 선택
4. 요청 양식 작성:
   - 퇴직 예정자: 박퇴사
   - 직급: 과장
   - 부서: 영업
   - 퇴직일: 2025-04-30
5. 제출 버튼 클릭
6. 시스템이 자동으로 티켓 생성 및 상태를 '처리 중'으로 변경
7. 자동화 규칙에 의해 해당 사용자의 자산 목록이 티켓에 첨부됨

#### 단계 2: 자산 확인 및 회수 계획
1. 서비스 요청 처리팀이 티켓 확인
2. 자동 생성된 자산 회수 체크리스트 검토
3. 할당된 자산 목록 확인:
   - 노트북: GS-PC-0006 (Dell Latitude 5420)
   - 태블릿: GS-TAB-0003 (iPad Air)
   - 모바일: GS-MOB-0006 (iPhone 13)
4. 회수 일정 조율: 2025-04-30 오전
5. 작업 상태 업데이트: "자산 회수 일정 조율 완료, 회수 대기 중"

#### 단계 3: 자산 회수 및 처리
1. 퇴직일에 모든 자산 회수 진행
2. 자산 상태 확인:
   - 노트북: 양호, 재사용 가능
   - 태블릿: 화면 균열, 수리 필요
   - 모바일: 양호, 재사용 가능
3. 데이터 백업 및 초기화 수행
4. 체크리스트 항목 완료 표시

#### 단계 4: 자산 정보 업데이트
1. 자산 정보 업데이트:
   - 할당 상태: 미할당
   - 사용자: 제거
   - 위치: IT 자산 창고
2. 노트북 및 모바일 상태를 '미사용'으로 변경
3. 태블릿 상태를 '수리 중'으로 변경
4. 태블릿 수리 요청 티켓 생성 (연계 티켓)

#### 단계 5: 완료 처리
1. '상태 변경'을 클릭하여 '해결됨'으로 변경
2. 해결 메모 입력: "박퇴사 과장 오프보딩 완료. 모든 자산 회수 및 초기화 완료. 태블릿(GS-TAB-0003)은 화면 수리 필요하여 수리 요청 티켓(#INC-789) 생성함."
3. 자동화 규칙에 의해 정상 자산 상태가 '미사용'으로 변경됨
4. 관리자가 완료 확인 후 '종료' 처리

### 시나리오 3: 자산 이관

#### 단계 1: 이관 요청 생성
1. 부서 관리자가 서비스 포털에 로그인
2. 상단의 '새 티켓' 버튼 클릭
3. 서비스 카탈로그에서 '자산 관리' > '자산 이관 요청' 선택
4. 요청 양식 작성:
   - 이관할 자산: GS-PC-0008 (Lenovo ThinkPad X1)
   - 현재 사용자: 임수진
   - 새 사용자: 정다운
   - 이관 사유: "임수진 대리 부서 이동에 따른 자산 이관"
   - 희망 이관일: 2025-04-20
5. 제출 버튼 클릭
6. 시스템이 자동으로 티켓 생성 및 상태를 '승인 대기'로 변경

#### 단계 2: 요청 승인 및 계획
1. IT 자산 관리자가 이관 요청 확인
2. '승인' 버튼 클릭
3. 시스템이 자동으로 상태를 '처리 중'으로 변경하고 서비스 요청 처리팀에 할당
4. 서비스 요청 처리팀이 이관 계획 수립:
   - 이관일: 2025-04-20
   - 담당자: 서비스 요청 처리팀 김담당
   - 필요 작업: 데이터 백업, 사용자 프로필 변경

#### 단계 3: 자산 이관 실행
1. 이관일에 현 사용자로부터 자산 회수
2. 중요 데이터 백업 및 필요시 초기화
3. 사용자 프로필 설정 변경
4. 새 사용자에게 자산 인계
5. 기본 사용법 안내 및 인수 확인서 서명 수령

#### 단계 4: 자산 정보 업데이트 및 완료
1. 자산 정보 업데이트:
   - 사용자: 정다운
   - 부서: 마케팅
   - 위치: 본사 2층
2. '상태 변경'을 클릭하여 '해결됨'으로 변경
3. 해결 메모 입력: "Lenovo ThinkPad X1(GS-PC-0008) 이관 완료. 임수진 대리에서 정다운 대리로 이관 완료."
4. 사용자가 완료 확인 후 '종료' 처리

### 시나리오 4: 자산 폐기

#### 단계 1: 폐기 요청 생성
1. IT 자산 담당자가 서비스 포털에 로그인
2. 상단의 '새 티켓' 버튼 클릭
3. 서비스 카탈로그에서 '자산 관리' > '자산 폐기 요청' 선택
4. 요청 양식 작성:
   - 폐기 대상 자산: GS-PC-0002 (HP EliteBook 840)
   - 폐기 사유: "노후화로 인한 성능 저하, 수리 비용 대비 효율성 낮음"
   - 대체 자산: GS-PC-0020 (HP EliteBook 860)
5. 제출 버튼 클릭
6. 시스템이 자동으로 티켓 생성 및 상태를 '승인 대기'로 변경

#### 단계 2: 승인 프로세스
1. IT 자산 관리자가 폐기 요청 확인
2. '승인' 버튼 클릭
3. 시스템이 자동으로 상태를 '처리 중'으로 변경
4. 자산 상태가 '폐기 예정'으로 자동 변경됨

#### 단계 3: 폐기 처리
1. 서비스 요청 처리팀이 폐기 처리 진행:
   - 자산의 저장 매체 데이터 완전 삭제
   - 라이선스 및 계정 회수
   - 인증된 폐기 업체를 통한 처리
   - 폐기 증명서 수령

#### 단계 4: 완료 처리
1. '상태 변경'을 클릭하여 '해결됨'으로 변경
2. 해결 메모 입력: "HP EliteBook 840(GS-PC-0002) 폐기 완료. 데이터 완전 삭제 및 환경 규제에 맞게 처리 완료. 폐기 증명서(#DC-456) 첨부."
3. 폐기 증명서 첨부
4. 자동화 규칙에 의해 자산 상태가 '폐기 완료'로 변경됨
5. 관리자가 완료 확인 후 '종료' 처리

## 7. 모범 사례 및 팁

### 자산 관리 최적화
- 정기적인 자산 인벤토리 감사 실시 (분기별 권장)
- 자산 태그 및 바코드 시스템 활용
- 자산 수명주기 단계별 명확한 프로세스 문서화
- 자산 사용률 및 성능 모니터링을 통한 최적화

### 보고 및 분석
- **자산 현황 대시보드 구성:**
  - 자산 유형별 분포
  - 부서별 자산 할당 현황
  - 상태별 자산 수량
  
- **수명주기 관련 보고서:**
  - 보증 만료 예정 자산 목록
  - 노후화 자산 교체 계획
  - 자산 평균 수명 및 비용 분석

- **핵심 성과 지표(KPI) 대시보드:**
  - 자산 활용률 (사용 중인 자산 / 전체 자산)
  - 미사용 자산 비율 (미사용 자산 / 전체 자산)
  - 자산별 TCO (총 소유 비용)
  - 자산 투자 수익률 (ROI)
  
- **커스텀 보고서 템플릿:**
  - **자산 상태 보고서** - 상태별 자산 수량 및 비율 표시
  - **보증 만료 알림 보고서** - 30/60/90일 내 보증 만료 예정 자산
  - **부서별 자산 할당 보고서** - 부서별 자산 유형 및 수량
  - **미할당 자산 보고서** - 사용 가능한 미할당 자산 목록

- **보고서 생성 방법:**
  1. 관리자 > 보고서 > 새 보고서 생성
  2. 보고서 유형 선택 (자산 보고서)
  3. 필터 및 그룹화 설정
  4. 표시할 필드 선택
  5. 저장 및 예약 설정

### 자동화 활용 팁
- 자산 상태 변경 시 자동 알림 설정
- 정기적인 자산 상태 보고서 자동 발송
- 보증 만료 전 알림 자동화로 갱신 누락 방지
- 입퇴사 프로세스와 자산 관리 워크플로우 통합

### 보안 및 규정 준수
- 자산 데이터 백업 정책 수립
- 민감한 데이터가 포함된 자산의 폐기 절차 강화
- 라이선스 규정 준수 모니터링
- 자산 관련 감사 추적 기록 유지
