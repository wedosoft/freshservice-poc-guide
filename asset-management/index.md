# 자산관리 워크플로우 템플릿

## 1. 자산관리 개요

### 목적
GS의 IT 자산 수명주기를 Freshservice를 통해 효율적으로 관리하고, 자산 관련 프로세스(입사-퇴직, 자산이관, 폐기 등)를 자동화하여 자산 가시성을 높이고 비용을 최적화하기 위한 워크플로우입니다.

### 자산 수명주기
1. 자산 도입 및 등록
2. 자산 할당 (신규 입사자 온보딩)
3. 자산 사용 및 유지보수
4. 자산 이관 (부서 이동, 사용자 변경)
5. 자산 회수 (퇴직자 오프보딩)
6. 자산 폐기 및 처분

## 2. 자산 데이터 구조

### 기본 정보
- **자산 태그**: 고유 식별자 (예: GS-PC-0001)
- **자산명**: 자산의 이름 (예: Dell Latitude 7420)
- **자산 유형**: PC, 노트북, 태블릿, 모니터 등
- **제조사**: Dell, HP, Apple 등
- **모델**: 제품 모델명
- **시리얼 번호**: 제조사 시리얼 번호

### 관리 정보
- **구매일**: 자산 구매 날짜
- **보증 만료일**: 보증 서비스 종료 날짜
- **상태**: 사용 중, 수리 중, 폐기 예정, 폐기 완료 등
- **할당 상태**: 할당됨, 미할당, 예약됨 등
- **위치**: 자산의 물리적 위치 (사무실, 지점 등)

### 할당 정보
- **사용자**: 자산을 사용하는 직원
- **부서**: 자산이 할당된 부서
- **할당일**: 사용자에게 할당된 날짜
- **반납 예정일**: 임시 할당의 경우 반납 예정 날짜

### 기술 정보
- **OS**: 운영체제 정보
- **CPU**: 프로세서 정보
- **RAM**: 메모리 용량
- **스토리지**: 저장 공간 정보
- **네트워크**: MAC 주소, IP 주소 등

## 4. 자산 수명주기 워크플로우

### 4.1 신규 입사자 자산 할당 워크플로우

#### 워크플로우 개요
신규 입사자에게 필요한 IT 자산을 준비하고 할당하는 프로세스입니다.

#### 워크플로우 단계
1. **입사 예정 알림**
   - 인사팀에서 신규 입사자 정보 입력
   - 입사 예정일 14일 전 자동 알림 생성

2. **자산 준비**
   - IT팀에서 필요 자산 목록 확인 (직무별 표준 구성)
   - 가용 자산 확인 또는 신규 자산 구매 요청
   - 자산 초기 설정 (OS 설치, 소프트웨어 설치, 계정 생성 등)

3. **자산 할당**
   - 자산 정보 업데이트 (할당 상태, 사용자, 부서 등)
   - 자산 인계 일정 조율

4. **자산 인계 및 교육**
   - 입사일에 자산 인계
   - 기본 사용법 및 IT 정책 안내
   - 자산 인수 확인서 서명

5. **완료 및 기록**
   - 자산 할당 완료 상태 업데이트
   - 관련 문서 첨부 및 기록

#### 구현 방법
1. 관리자 > 작업 흐름 자동화 도구에서:
   - 새 자동화 규칙 생성
   - 트리거 탭에서:
     - 이벤트: 신규 입사자 온보딩 요청 생성
   - 작업 탭에서:
     - 자산 준비 체크리스트 생성
     - 담당자 할당
     - 알림 설정
   - 저장 및 활성화

### 4.2 퇴직자 자산 회수 워크플로우

#### 워크플로우 개요
퇴직 예정 직원의 IT 자산을 회수하고 적절히 처리하는 프로세스입니다.

#### 워크플로우 단계
1. **퇴직 예정 알림**
   - 인사팀에서 퇴직 예정자 정보 입력
   - 퇴직 예정일 14일 전 자동 알림 생성

2. **자산 확인**
   - 할당된 모든 자산 목록 확인
   - 자산 회수 체크리스트 생성

3. **자산 회수**
   - 퇴직일에 모든 자산 회수
   - 자산 상태 확인 및 기록
   - 데이터 백업 및 초기화

4. **자산 상태 업데이트**
   - 자산 정보 업데이트 (할당 상태, 사용자 제거 등)
   - 자산 위치 업데이트 (IT 창고 등)

5. **자산 평가 및 처리 결정**
   - 자산 상태 평가
   - 재사용, 수리, 폐기 등 처리 방향 결정

#### 구현 방법
1. 관리자 > 작업 흐름 자동화 도구에서:
   - 새 자동화 규칙 생성
   - 트리거 탭에서:
     - 이벤트: 퇴직자 오프보딩 요청 생성
   - 작업 탭에서:
     - 자산 회수 체크리스트 생성
     - 담당자 할당
     - 알림 설정
   - 저장 및 활성화

### 4.3 자산 이관 워크플로우

#### 워크플로우 개요
부서 이동이나 업무 변경 등으로 인해 자산을 다른 사용자에게 이관하는 프로세스입니다.

#### 워크플로우 단계
1. **이관 요청**
   - 부서 관리자 또는 사용자가 자산 이관 요청 생성
   - 이관 사유, 대상 사용자, 희망일 등 정보 입력

2. **요청 승인**
   - IT 관리자가 요청 검토 및 승인
   - 이관 계획 수립

3. **자산 회수 및 준비**
   - 현 사용자로부터 자산 회수
   - 필요 시 데이터 백업 및 초기화
   - 새 사용자를 위한 설정 변경

4. **자산 재할당**
   - 새 사용자에게 자산 인계
   - 기본 사용법 안내 (필요 시)
   - 자산 인수 확인서 서명

5. **완료 및 기록**
   - 자산 정보 업데이트 (사용자, 부서, 위치 등)
   - 관련 문서 첨부 및 기록

#### 구현 방법
1. 관리자 > 서비스 카탈로그에서:
   - 새 서비스 항목 생성 (자산 이관 요청)
   - 양식 필드 구성
   - 작업 흐름 자동화 규칙 설정
   - 저장 및 게시

### 4.4 자산 폐기 워크플로우

#### 워크플로우 개요
노후화되거나 손상된 자산을 안전하게 폐기하는 프로세스입니다.

#### 워크플로우 단계
1. **폐기 대상 식별**
   - 노후 자산 보고서 검토
   - 수리 불가능한 손상 자산 확인
   - 폐기 대상 목록 작성

2. **폐기 승인 요청**
   - 폐기 대상 자산 목록 및 사유 제출
   - 대체 자산 계획 포함 (필요 시)

3. **승인 프로세스**
   - IT 관리자 승인
   - 자산 관리 책임자 승인
   - 필요 시 재무 부서 승인

4. **데이터 삭제 및 초기화**
   - 저장 매체의 데이터 완전 삭제
   - 공장 초기화 또는 하드 드라이브 물리적 파괴

5. **물리적 폐기**
   - 환경 규제에 맞는 폐기 방법 선택
   - 인증된 폐기 업체 활용
   - 폐기 증명서 수령

6. **자산 기록 업데이트**
   - 자산 상태를 '폐기 완료'로 변경
   - 폐기 날짜, 방법, 증명서 번호 등 기록
   - 관련 문서 첨부

#### 구현 방법
1. 관리자 > 서비스 카탈로그에서:
   - 새 서비스 항목 생성 (자산 폐기 요청)
   - 양식 필드 구성
   - 작업 흐름 자동화 규칙 설정
   - 저장 및 게시

## 5. 구현 단계

### 5.1 자산 데이터 업로드

#### CSV 파일 준비
1. 제공된 샘플 CSV 형식에 맞게 자산 데이터 준비
2. 필수 필드 확인 (자산 태그, 자산명, 자산 유형, 제조사, 모델 등)
3. 날짜 형식 통일 (YYYY-MM-DD)

#### 업로드 절차
1. 관리자 > 자산 > 가져오기에서:
   - CSV 가져오기 선택
   - 준비된 CSV 파일 업로드
   - 필드 매핑 확인 및 조정
   - 가져오기 실행
   - 결과 확인 및 오류 수정

### 5.2 자산 관계 설정

#### 관계 유형 정의
1. 관리자 > 자산 > 관계 유형에서:
   - 필요한 관계 유형 정의:
     - 연결됨 (예: PC와 모니터)
     - 부속품 (예: 노트북과 도킹 스테이션)
     - 하위 항목 (예: 워크스테이션과 그래픽 카드)

#### 자산 간 관계 설정
1. 자산 > 자산 목록에서:
   - 관계를 설정할 자산 선택
   - '관계' 탭 선택
   - '관계 추가' 버튼 클릭
   - 관련 자산 및 관계 유형 선택
   - 저장

### 5.3 자산 수명주기 자동화 설정

#### 자산 상태 정의
1. 관리자 > 자산 > 자산 상태에서:
   - 필요한 상태 정의:
     - 사용 중
     - 미사용
     - 수리 중
     - 폐기 예정
     - 폐기 완료

#### 자동화 규칙 설정
1. 관리자 > 작업 흐름 자동화 도구에서:
   - 새 자동화 규칙 생성
   - 자산 수명주기 이벤트에 따른 규칙 설정:
     - 신규 자산 등록 시 알림
     - 보증 만료 30일 전 알림
     - 자산 할당 시 사용자에게 자동 알림
     - 자산 회수 시 상태 자동 변경
   - 저장 및 활성화

### 5.4 서비스 요청 템플릿 설정

#### 온보딩/오프보딩 템플릿
1. 관리자 > 서비스 카탈로그에서:

     - 필요 자산 목록 (직무별 표준 구성 선택)
     - 필요 소프트웨어 목록
   - 오프보딩 템플릿 구성:
     - 퇴직 예정자 정보 (이름, 직급, 부서, 퇴직일 등)
     - 할당된 자산 목록 자동 표시
     - 데이터 처리 지침
   - 저장 및 게시

#### 자산 이관/폐기 템플릿
1. 관리자 > 서비스 카탈로그에서:
   - 새 서비스 항목 생성
   - 자산 이관 템플릿 구성:
     - 이관할 자산 선택
     - 현재 사용자 정보
     - 새 사용자 정보
     - 이관 사유 및 희망일
   - 자산 폐기 템플릿 구성:
     - 폐기할 자산 선택
     - 폐기 사유
     - 대체 자산 계획 (필요 시)
   - 저장 및 게시
