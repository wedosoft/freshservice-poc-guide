# 자산관리 워크플로우

> **실습 링크**: <a href="https://gsholdings.freshservice.com/cmdb/items" target="_blank">자산 관리 화면</a>

## 1. 자산 수명주기 개요

Freshservice에서 관리 가능한 자산 수명주기:
- 자산 도입 및 등록 → 자산 할당 → 사용 및 유지보수 → 자산 이관 → 자산 회수 → 자산 폐기

## 2. 설정 및 테스트 방법

| 설정 항목 | 실습 링크 | 주요 기능 |
|---------|-----------|---------|
| 자산 가져오기 | <a href="https://gsholdings.freshservice.com/cmdb/items" target="_blank">데이터 업로드</a> | CSV 파일로 자산 데이터 대량 등록 |
| 자산 유형 관리 | <a href="https://gsholdings.freshservice.com/cmdb/ci_types" target="_blank">유형 설정</a> | 하드웨어, 소프트웨어 등 유형별 속성 정의 |
| 자산 자동화 | <a href="https://gsholdings.freshservice.com/ws/2/admin/automators" target="_blank">워크플로우 설정</a> | 자산 상태 변경, 알림 자동화 |

### 3.1 자산관리 구성 단계

아래 단계를 따라 자산관리 시스템을 구성합니다:

#### 1단계: 자산 유형 설정 (소요 시간: 20분)
1. 관리자 메뉴에서 '자산' > '자산 유형' 선택
2. 다음 자산 유형을 구성:
   - **노트북/PC 자산 유형**
     - 기본 정보: 이름, 자산 태그, 시리얼 번호, 제조사, 모델
     - 확장 정보: OS, CPU, RAM, 스토리지, 구매일, 보증기간
     - 상태 정보: 상태(신규, 사용 중, 수리 중, 폐기 예정, 폐기됨)
   
   - **모바일/태블릿 자산 유형**
     - 기본 정보: 이름, 자산 태그, 시리얼 번호, 제조사, 모델
     - 확장 정보: OS, 통신사, 요금제, 구매일, 보증기간
     - 상태 정보: 상태(신규, 사용 중, 수리 중, 폐기 예정, 폐기됨)
   
   - **주변기기 자산 유형**
     - 기본 정보: 이름, 자산 태그, 시리얼 번호, 제조사, 모델
     - 확장 정보: 종류(모니터, 키보드, 마우스 등), 구매일, 보증기간
     - 상태 정보: 상태(신규, 사용 중, 수리 중, 폐기 예정, 폐기됨)

3. 각 자산 유형에 사용자 정의 필드 추가:
   - 담당부서 (문자열)
   - 비용센터 (문자열)
   - 자산분류 (드롭다운: 필수자산, 선택자산)

#### 2단계: 자산 데이터 가져오기 (소요 시간: 20분)
1. 위의 샘플 CSV 데이터를 다운로드하거나 복사하여 새 CSV 파일 생성
2. CSV 파일을 UTF-8 인코딩으로 저장
3. 관리자 메뉴에서 '자산' > '가져오기' 선택
4. '파일 선택' 버튼을 클릭하여 CSV 파일 업로드
5. 필드 매핑 확인 (한글 칼럼명 → 영문 필드명 매핑)
   - 자산_태그 → Asset Tag
   - 자산명 → Name
   - 자산_유형 → Asset Type
   - 제조사 → Manufacturer
   - 모델 → Model
   - 기타 필드도 적절히 매핑
6. '가져오기' 버튼 클릭하여 진행

#### 3단계: 자산 자동화 규칙 설정 (소요 시간: 15분)
1. 관리자 메뉴에서 '작업 흐름 자동화 도구' 선택
2. '새 규칙 추가' 버튼 클릭하여 자산 관련 자동화 규칙 생성:

   **규칙 1: 새 자산 등록 시 IT 담당자 알림**
   - 이벤트: 새 자산 생성 시
   - 조건: 모든 자산
   - 작업: 이메일 알림 (대상: IT 담당자)

   **규칙 2: 자산 상태 변경 시 기록**
   - 이벤트: 자산 업데이트 시
   - 조건: 상태 변경됨 = 예
   - 작업: 내부 노트 생성 (내용: 자산 상태가 ${old_status}에서 ${new_status}로 변경됨)

   **규칙 3: 보증 만료 예정 알림**
   - 이벤트: 예약 이벤트 (매주 월요일)
   - 조건: 보증 만료일 < 현재일+30일
   - 작업: 이메일 알림 (대상: IT 자산 관리자)

## 4. 주요 테스트 시나리오

> **실습 링크**: <a href="https://gsholdings.freshservice.com/support/tickets/new" target="_blank">새 티켓 생성 화면</a>

| 시나리오 | 테스트 방법 |
|---------|------------|
| 신규 입사자 자산 할당 | 입사자 정보 입력 → 자산 준비 → 자산 할당 |
| 퇴직자 자산 회수 | 퇴직자 정보 입력 → 자산 목록 확인 → 자산 회수 |
| 자산 이관 | 이관 요청 → 데이터 백업 → 재할당 → 정보 업데이트 |
| 자산 폐기 | 폐기 대상 식별 → 승인 요청 → 상태 변경 |

### 4.1 신규 입사자 자산 할당 시나리오 상세 단계

1. **입사자 정보 등록**
   - 관리자 메뉴에서 '사용자' > '요청자' 선택
   - '새 요청자 추가' 버튼 클릭
   - 신규 입사자 정보 입력 (예: 김신입, 개발팀, 직위: 사원)
   - 저장 버튼 클릭

2. **자산 할당 티켓 생성**
   - 서비스 포털에서 '새 티켓' 버튼 클릭
   - 제목: "김신입 입사자 장비 할당 요청"
   - 요청자: 부서 관리자(예: 개발팀장)
   - 우선순위: 중간
   - 설명: "5월 15일 입사 예정인 김신입 사원을 위한 PC 및 모바일 기기 준비 요청"
   - 제출 버튼 클릭

3. **자산 할당 처리**
   - IT 담당자로 로그인하여 할당 요청 티켓 확인
   - '자산' 탭으로 이동
   - '자산 연결' 버튼 클릭
   - 사용 가능한 자산 목록에서 적절한 노트북 선택 (예: GS-PC-0003)
   - 관계 유형: '할당 대상'으로 설정
   - 저장 버튼 클릭
   - 자산 상태를 '할당됨'으로 변경
   - 사용자 필드에 '김신입' 설정
   - 할당일 필드에 현재 날짜 설정
   - 저장 버튼 클릭

4. **할당 완료 처리**
   - 티켓에 내부 노트 추가: "노트북 및 주변기기 할당 완료"
   - 연결된 자산 목록 확인
   - 티켓 상태를 '해결됨'으로 변경
   - 해결 노트: "신규 입사자 김신입에게 노트북(GS-PC-0003) 할당 완료"
   - 저장 버튼 클릭

### 4.2 퇴직자 자산 회수 시나리오 상세 단계

1. **퇴직 처리 티켓 생성**
   - 서비스 포털에서 '새 티켓' 버튼 클릭
   - 제목: "이철수 퇴직자 자산 회수"
   - 요청자: 부서 관리자(예: 영업팀장)
   - 우선순위: 중간
   - 설명: "5월 30일 퇴직 예정인 이철수 사원의 모든 회사 자산 회수 요청"
   - 제출 버튼 클릭

2. **사용자 자산 조회**
   - IT 담당자로 로그인하여 티켓 확인
   - '자산' 메뉴로 이동
   - 검색 필드에 "이철수" 입력하여 할당된 모든 자산 조회
   - 확인된 자산(GS-PC-0003, GS-TAB-0002)을 티켓에 연결

3. **자산 회수 처리**
   - 각 자산의 상태를 '회수됨'으로 변경
   - 사용자 필드를 비움
   - 할당 상태를 '할당 해제됨'으로 변경
   - 내부 노트에 회수 상태 기록 (예: "하드웨어 점검 완료, 양호한 상태로 회수")
   - 저장 버튼 클릭

4. **회수 완료 처리**
   - 티켓에 내부 노트 추가: "모든 자산 회수 완료, 하드웨어 상태 양호"
   - 티켓 상태를 '해결됨'으로 변경
   - 해결 노트: "이철수 사원의 노트북 및 태블릿 회수 완료"
   - 저장 버튼 클릭

### 4.3 자산 폐기 시나리오 상세 단계

1. **폐기 대상 식별**
   - '자산' 메뉴로 이동
   - 검색 필터: 구매일 < (현재년도-4년) (4년 이상 된 자산)
   - 검색 결과에서 폐기 대상 자산 선택 (예: GS-PC-0002)

2. **폐기 승인 요청**
   - 서비스 포털에서 '새 티켓' 버튼 클릭
   - 제목: "노후 PC 폐기 승인 요청"
   - 요청자: IT 담당자
   - 우선순위: 낮음
   - 설명: "사용 연한이 지난 PC의 폐기 승인 요청"
   - 해당 자산을 티켓에 연결
   - 승인자: IT 관리자
   - 제출 버튼 클릭

3. **승인 처리**
   - IT 관리자로 로그인
   - 승인 요청 확인 및 '승인' 버튼 클릭
   - 승인 사유: "사용 연한 경과로 인한 폐기 승인"

4. **폐기 처리**
   - IT 담당자로 로그인
   - 승인된 티켓 확인
   - 자산 상태를 '폐기 예정'으로 변경
   - 폐기 예정일 설정
   - 내부 노트: "데이터 완전 삭제 처리 완료"
   - 자산 상태를 최종적으로 '폐기됨'으로 변경
   - 폐기일 필드에 현재 날짜 설정
   - 저장 버튼 클릭
   - 티켓 상태를 '해결됨'으로 변경
   - 해결 노트: "지정된 자산 폐기 처리 완료"
   - 저장 버튼 클릭

## 5. 확인 포인트

- 자산 데이터 가져오기 성공 여부
- 자산과 티켓 연결 기능
- 자산 상태 자동 업데이트
- 자산 보고서 생성 및 대시보드 구성

### 5.1 자산 보고서 생성 방법

1. 관리자 메뉴에서 '보고서' 선택
2. '새 보고서 추가' 버튼 클릭
3. 다음 보고서 생성:
   - **부서별 자산 현황 보고서**
     - 보고서 유형: 자산 보고서
     - 그룹화 기준: 부서
     - 시각화 유형: 막대 그래프
   
   - **자산 유형별 분포 보고서**
     - 보고서 유형: 자산 보고서
     - 그룹화 기준: 자산 유형
     - 시각화 유형: 파이 차트
   
   - **만료 예정 보증 보고서**
     - 보고서 유형: 자산 보고서
     - 필터: 보증 만료일 < 현재일+90일
     - 정렬: 보증 만료일 (오름차순)
     - 시각화 유형: 표

4. 각 보고서를 대시보드에 추가
   - '대시보드' 메뉴 선택
   - '새 대시보드' 버튼 클릭
   - 이름: "자산 관리 대시보드"
   - 생성한 각 보고서를 대시보드에 추가
   - 저장 버튼 클릭

### 5.2 실행 결과 확인사항

- **데이터 정확성**: 가져온 자산 데이터가 정확하게 반영되었는지 확인
- **관계 설정**: 자산-사용자, 자산-티켓 관계가 올바르게 설정되었는지 확인
- **상태 변경**: 자산 수명주기별 상태 변경이 잘 작동하는지 확인
- **자동화 규칙**: 설정한 자동화 규칙이 예상대로 작동하는지 확인
- **보고서**: 자산 데이터가 보고서와 대시보드에 정확히 반영되는지 확인
